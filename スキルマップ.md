# スキルマップ

組み込みソフトウェア開発チームにおける役割別のスキル定義と習熟度目標。現状把握・ギャップ分析・育成計画の基礎として使用する。

---

## 1. 習熟度レベルの定義

| レベル | 名称 | 定義 |
|-------|------|------|
| **0** | 未経験 | 知らない・やったことがない |
| **1** | 見習い | 手順書・テンプレートがあれば一人でできる |
| **2** | 自立 | 手順書なしで一人でできる。標準的なケースに対応できる |
| **3** | 熟練 | 例外ケースにも対応できる。他者に教えられる・改善提案できる |

## 4. ロール別のスキル優先順位

### 開発者が最初に身につけるべきスキル（Lv1到達目標）

改善活動の初期（Month 1〜2）に最低限機能する状態として、以下を最優先とする。

| 優先 | 項目 | 理由 |
|------|------|------|
| ★★★ | 6-2 進捗報告 | チームの状況把握の基本 |
| ★★★ | 7-2 エスカレーション | 不明なまま進める手戻りを防ぐ |
| ★★★ | 5-1 バージョン管理の基本操作 | 構成管理の最低限 |
| ★★☆ | 1-1 開発プロセスの全体像 | 自分が何工程にいるか分かる |
| ★★☆ | 4-5 レビューへの参加 | 品質の最低限 |

### PLが最初に身につけるべきスキル（Lv2到達目標）

改善活動が機能し始める（Month 3〜4）ために必要なスキル。

| 優先 | 項目 | 理由 |
|------|------|------|
| ★★★ | 6-5 課題管理 | チームのタスクを可視化する |
| ★★★ | 6-3 遅延・ブロックの検知 | 問題を早期に捕まえる |
| ★★★ | 5-3 変更影響の分析 | 手戻りを防ぐ |
| ★★☆ | 1-2 工程の入口・出口基準 | 完了の曖昧さを排除する |
| ★★☆ | 7-5 チーム横断の調整 | チーム間の認識齟齬を防ぐ |

### 推進者が最初に身につけるべきスキル（Lv3到達目標）

改善活動を主導するために必要なスキル。

| 優先 | 項目 | 理由 |
|------|------|------|
| ★★★ | 8-2 根本原因分析 | 問題の本質を捉えないと施策が的外れになる |
| ★★★ | 7-4 上位者への報告 | 部長へのスポンサー合意に直結する |
| ★★★ | 1-2 工程の入口・出口基準 | ワークショップをファシリテートするために必要 |
| ★★☆ | 6-4 リスク管理 | 改善活動自体のリスクを管理する |
| ★★☆ | 7-5 合意形成 | 複数チームを動かすために必要 |

## 5. ギャップ分析の使い方

### ステップ1：現状評価の実施

- 自己評価と上長評価の両方を行う
- 評価の乖離が大きい項目は「認識のズレ」として議論する
- 評価は定性的な証拠（「この設計書を書いた」「このレビューで指摘した」）をセットで記録する

### ステップ2：ギャップの優先順位付け

以下の基準でギャップを優先する。

```
優先度 高（★★★）
  └ 現状Lv 0 かつ 目標Lv 2以上
  └ チームの進捗・品質に直接影響するスキル

優先度 中（★★☆）
  └ 現状Lv 1 かつ 目標Lv 3
  └ 改善サイクルが回り始めてから必要になるスキル

優先度 低（★☆☆）
  └ 現状Lv 2 かつ 目標Lv 3
  └ すでに自立しており熟練度を上げる段階
```

### ステップ3：育成アクションの設計

| ギャップの種類 | 育成アクションの例 |
|-------------|----------------|
| 知識不足（Lv0→1） | OJT・ドキュメント読み込み・勉強会への参加 |
| 実践不足（Lv1→2） | 実案件でのアサイン・ペアワーク・フィードバック付き実施 |
| 応用・指導力（Lv2→3） | 後輩へのレクチャー・改善提案の実行・ファシリテーション経験 |

### ステップ4：定期的な再評価

| タイミング | 実施内容 |
|----------|---------|
| 改善活動開始時 | 全員のベースライン評価を実施・優先ギャップを特定 |
| 3ヶ月後 | 育成アクションの効果確認・目標レベルの達成度確認 |
| 6ヶ月後 | 目標レベルの見直し・次フェーズのスキル目標設定 |

## 付録：スキルマップと改善フェーズの対応

改善フェーズの進行とともに、チームに求めるスキルレベルも段階的に引き上げる。

| フェーズ | 開発者に求める変化 | PLに求める変化 | 推進者の関与 |
|---------|-----------------|-------------|------------|
| **Phase 0〜1** | 6-2・7-2のLv1 | 6-5・6-3のLv2 | 8-2・7-4でリード |
| **Phase 2** | 1-1・1-2のLv1 | 1-2・5-3のLv2 | 1-2・7-5でリード |
| **Phase 3** | 4-5・5-1のLv2 | 8-1・8-3のLv2 | 8-4でメトリクス計測 |
| **Phase 4以降** | 8-1のLv1（自ら問題発見） | 全カテゴリLv2安定 | Lv3への引き上げ支援 |
