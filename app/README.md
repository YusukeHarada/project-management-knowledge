# スキル診断アプリ

組み込みSW開発チームのスキル自己評価 Web アプリ。スキルマップ（10カテゴリ・53項目）を Web 上で入力・集計し、個人とチームのダッシュボードで結果を確認できる。

## 起動方法

```bash
npm install    # 初回のみ
cd app && npm run dev    # http://localhost:3000
```

## コマンド一覧

```bash
npm run dev      # 開発サーバー起動
npm run build    # 本番ビルド
npm run lint     # ESLint
npm test         # ユニットテスト（Vitest）
```

## 画面構成

| 画面 | URL | 内容 |
|------|-----|------|
| トップ | `/` | 名前・ロール入力で診断開始 |
| スキル評価 | `/assessment` | 10カテゴリ・53項目を Lv0〜3 で評価 |
| 個人ダッシュボード | `/dashboard/[userId]` | レーダーチャート・ギャップ一覧・育成アクション |
| チームダッシュボード | `/dashboard/team` | カテゴリ別平均・弱点サマリ・ヒートマップ |
| マスタ管理 | `/admin` | カテゴリ・スキル項目・ロール別目標 Lv の編集 |

## 運用ガイド

### 1. SQLite ↔ Firestore の切り替え

#### ローカル開発

`.env.local` の `DB_BACKEND` を変更してサーバーを再起動するだけで切り替わる。

**SQLite モード（デフォルト・認証なし）**
```bash
# .env.local
DB_BACKEND=sqlite
NEXT_PUBLIC_DB_BACKEND=sqlite
```

**Firestore モード（Google 認証あり）**
```bash
# .env.local
DB_BACKEND=firestore
NEXT_PUBLIC_DB_BACKEND=firestore
NEXT_PUBLIC_FIREBASE_API_KEY=...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=...
NEXT_PUBLIC_FIREBASE_PROJECT_ID=...
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=...
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=...
NEXT_PUBLIC_FIREBASE_APP_ID=...
FIREBASE_SERVICE_ACCOUNT_KEY=...   # JSON を1行に
```

変更後は `npm run dev` を再起動する。

> **注意**: SQLite のデータと Firestore のデータは独立しており、切り替えても互いに引き継がれない。

#### Vercel（本番）

Vercel はサーバーレス環境でファイルシステムに書き込めないため、**Firestore モード一択**。

1. Vercel ダッシュボード → プロジェクト → **Settings → Environment Variables**
2. 上記の Firestore モード用の環境変数をすべて追加
3. デプロイを再実行（または `git push` でトリガー）

### 2. スキルマップを変更する手順

スキルマップの**唯一の変更先はアプリの管理画面**（`/admin`）。

#### カテゴリを追加・編集・削除する

1. `/admin` を開く
2. 「カテゴリ管理」タブを選択
3. 新規追加: 「カテゴリ名」を入力して「追加」
4. 削除: カテゴリ右の「削除」ボタン（関連スキル項目も同時に削除される）

#### スキル項目を追加・編集・削除する

1. `/admin` を開く
2. 「スキル項目管理」タブを選択
3. 「新規スキル項目を追加」ボタンをクリック
4. カテゴリ・番号・評価項目名・ロール別目標レベルを入力して「保存」
5. 既存項目の「編集」で目標レベルの変更も可能

#### スキルマップ.md に書き出す

管理画面での変更を `スキルマップ.md` に反映させたい場合：

1. `/admin` を開く
2. 画面右上の「**スキルマップ.md を更新する**」ボタンをクリック
3. `スキルマップ.md` が自動生成される

> Vercel 上では `スキルマップ.md` の書き出しはできない（ファイルシステムが読み取り専用のため）。ローカルで実行すること。

### 3. アプリの使い方・全体フロー

```
① トップページ（/）
   └ 「新規で始める」または「続きから診断する」を選択

② スキル評価（/assessment）
   └ 10カテゴリ・53項目を Lv0〜3 で評価
   └ 「結果を保存してダッシュボードへ」で完了

③ 個人ダッシュボード（/dashboard/[userId]）
   └ レーダーチャートでカテゴリ別の現状・目標を確認
   └ 優先育成アクション・全項目ギャップ一覧を確認

④ チームダッシュボード（/dashboard/team）
   └ チーム全体のレーダーチャート・弱点サマリ・ヒートマップを確認
```

### 4. スキル診断を登録する手順

#### 新規登録の場合

1. トップページ（`/`）を開く
2. 「**新規で始める**」タブを選択
3. お名前とロール（開発者 / PL / PM / 推進者）を入力
4. 「診断を開始する」をクリック
5. スキル評価画面でカテゴリごとに各項目の現状レベル（Lv0〜3）を選択
   - 根拠・エビデンスは任意入力（例：「○○設計書を作成」「○○レビューで指摘」）
   - カテゴリ末尾の「次のカテゴリへ →」で順番に進める
6. 最後のカテゴリまで入力したら「結果を保存してダッシュボードへ」をクリック

#### 再診断（既存ユーザー）の場合

1. トップページの「**続きから診断する**」タブを選択
2. 自分の名前をドロップダウンから選択
3. 「この人で診断を続ける」をクリック
4. 前回の回答が自動で反映された状態で評価画面が開く
5. 変更したい項目だけ修正して保存

または、個人ダッシュボード右上の「**再診断する**」リンクからも直接遷移できる。

---

### 5. 過去のスキルマップと現在を比較する

#### 個人ダッシュボードでの比較

1. 個人ダッシュボード（`/dashboard/[userId]`）を開く
2. 「全項目ギャップ一覧」の右上に「**前回比較**」ドロップダウンが表示される
   （診断を2回以上行っている場合のみ表示）
3. 比較したい過去の診断日時を選択
4. 「現状」列に `+1`（成長）・`-1`（低下）・`±0`（変化なし）が表示される

#### チームの経時変化を見る場合

現状ではチームの時系列比較機能は未実装。各メンバーが個人ダッシュボードで前回比を確認し、チームで共有するのが現時点でのワークフロー。

### レベル定義

| Lv | 名称 | 目安 |
|----|------|------|
| 0 | 未経験 | 知らない・やったことがない |
| 1 | 見習い | 手順書があれば一人でできる |
| 2 | 自立 | 手順書なしで一人でできる |
| 3 | 熟練 | 例外ケースにも対応・他者に教えられる |
