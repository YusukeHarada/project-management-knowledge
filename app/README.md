# スキル診断アプリ

組み込みSW開発チームのスキル自己評価 Web アプリ。10カテゴリ・53項目のスキルを自己評価し、個人とチームのダッシュボードで結果を確認できる。

## チームメンバー向け：スキル診断の使い方

### はじめてのログイン

1. アプリの URL にアクセスする
2. **「Google でサインイン」** をクリックし、会社または個人の Google アカウントでログイン
3. 初回のみ **表示名**（アプリ内での呼び名）と **ロール** を入力して登録する

> SQLite モード（ローカル環境）を使う場合はログイン不要。「新規で始める」タブから名前とロールを入力する。

---

### スキル診断を受ける

1. トップページの **「診断を開始する」** をクリック
2. 左のカテゴリメニューから各カテゴリを選択（または下部の「次のカテゴリへ →」で順番に進む）
3. 各スキル項目に対して **Lv0〜3** のいずれかを選択する

| レベル | 基準 |
|--------|------|
| **Lv0** 未経験 | やったことがない |
| **Lv1** 見習い | 手順書があれば一人でできる |
| **Lv2** 自立 | 手順書なしで一人でできる |
| **Lv3** 熟練 | 例外ケースにも対応でき、他者に教えられる |

4. 任意で **根拠・エビデンス**（「○○設計書を作成した」など）を入力する
5. すべて入力したら **「結果を保存してダッシュボードへ」** をクリック

---

### 診断結果を確認する（個人ダッシュボード）

診断後は個人ダッシュボードが表示される。

| セクション | 内容 |
|-----------|------|
| カテゴリ別スキルレーダー | 現状（青）と目標（緑）の比較チャート |
| 成長トレンド | 2回以上診断した場合、カテゴリ別の時系列グラフ |
| 優先育成アクション | ギャップが大きい項目と推奨アクション |
| 全項目ギャップ一覧 | 全53項目の現状・目標・ギャップ・優先度 |

---

### 過去の診断と比較する

全項目ギャップ一覧の右上に **「表示する過去の診断回数」** セレクタが表示される（2回以上診断した場合）。

- 1〜5回分を選択すると、各回のレベルが列として並んで表示される
- 隣接する回との差分（+1 / -1 / ±0）も各セルに表示される

---

### 再診断する・過去の診断を修正する

**再診断**（新しい結果として保存）
- 個人ダッシュボード右上の **「再診断する」** をクリック
- 前回の入力値が自動でセットされるので、変化した項目だけ修正して保存

**過去の診断を修正**（記載ミスを直したいとき）
- 全項目ギャップ一覧の各セッション列の日付の下にある **「編集」** をクリック
- その回のデータが読み込まれた状態で評価フォームが開く
- 修正して保存すると、同じ診断回のデータが上書きされる（新しい回は増えない）

**診断回の削除**
- 各セッション列の日付の下にある **「削除」** をクリック
- 確認後にその回の診断データが削除される

---

### チームダッシュボードを見る

ヘッダーの **「チームダッシュボード」** から全員の集計を確認できる。

| セクション | 内容 |
|-----------|------|
| チームスキルレーダー | チーム全体の平均現状・平均目標の比較チャート |
| カテゴリ別チーム平均 | 10カテゴリの平均スコア（赤=1未満、黄=2未満、青=2以上） |
| チームの弱点 | 平均 Lv1 未満の項目 |
| スキルヒートマップ | 全項目×全メンバーのレベル一覧表（カテゴリ絞り込み可） |

---

### 設定を変更する（表示名・ロール・アカウント削除）

1. ヘッダー右の **「設定」** をクリック（または個人ダッシュボードの「設定」ボタン）
2. 表示名・ロールを変更して **「変更を保存する」** をクリック
3. アカウントを削除したい場合は **「アカウントを削除する」** をクリック（診断データも全削除される）

## 管理者向け：マスタ管理とユーザー管理

管理者のみヘッダーに **「マスタ管理」** リンクが表示される。管理者の設定は開発者が行う（後述）。

### スキル項目を追加・編集・削除する

1. **マスタ管理** → **「スキル項目管理」** タブを選択
2. 「新規スキル項目を追加」ボタンからカテゴリ・番号・評価項目名を入力
3. 各ロール（開発者/PL/PM/推進者）の目標レベルを設定して保存
4. 既存項目の「編集」「削除」も同画面から操作

### カテゴリを追加・削除する

1. **マスタ管理** → **「カテゴリ管理」** タブを選択
2. カテゴリ名を入力して「追加」
3. カテゴリを削除すると、そのカテゴリ内のスキル項目も一括削除される

### ユーザーを管理する

1. **マスタ管理** → **「ユーザー管理（管理者）」** タブを選択
2. 登録済みユーザーの一覧が表示される
3. 「ダッシュボード」で各ユーザーの診断結果を確認できる
4. 「削除」でユーザーと全診断データを削除できる（元に戻せない）

### スキルマップ.md を更新する

マスタ管理画面での変更内容をリポジトリの `スキルマップ.md` に書き出したい場合：

1. **マスタ管理** 画面右上の **「スキルマップ.md を更新する」** をクリック
2. `スキルマップ.md` が自動生成・上書きされる

> この操作はローカル環境でのみ有効。Vercel 上ではファイルシステムへの書き込みができない。

## 開発者・管理者向け：セットアップと運用

### 起動方法

```bash
cd app
npm install    # 初回のみ
npm run dev    # http://localhost:3000
```

### コマンド一覧

```bash
npm run dev      # 開発サーバー起動
npm run build    # 本番ビルド確認
npm run lint     # ESLint
npm test         # ユニットテスト（Vitest）
```

---

### データベースの切り替え（SQLite / Firestore）

`app/.env.local` の `DB_BACKEND` で切り替える。変更後は `npm run dev` を再起動する。

**SQLite モード（デフォルト・認証なし）**
```bash
DB_BACKEND=sqlite
NEXT_PUBLIC_DB_BACKEND=sqlite
```

**Firestore モード（Google 認証あり・本番推奨）**
```bash
DB_BACKEND=firestore
NEXT_PUBLIC_DB_BACKEND=firestore
NEXT_PUBLIC_FIREBASE_API_KEY=...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=...
NEXT_PUBLIC_FIREBASE_PROJECT_ID=...
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=...
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=...
NEXT_PUBLIC_FIREBASE_APP_ID=...
FIREBASE_SERVICE_ACCOUNT_KEY=...   # サービスアカウント JSON を1行化したもの
```

> SQLite のデータと Firestore のデータは独立している。切り替えてもデータは引き継がれない。

---

### Firebase のセットアップ（Firestoreモード）

1. [Firebase Console](https://console.firebase.google.com/) でプロジェクトを作成
2. **Authentication** → Google プロバイダーを有効化
3. **Firestore Database** → データベースを作成（リージョン: `asia-northeast1` 推奨）
4. **プロジェクト設定** → サービスアカウント → 秘密鍵を生成してダウンロード
5. `.env.local` に Firebase の設定値を記入

---

### Vercel へのデプロイ

1. Vercel でプロジェクトを作成し、このリポジトリと連携
2. **Settings → General → Root Directory** を `app` に設定
3. **Settings → Environment Variables** に以下を追加：
   - Firestore モードの環境変数をすべて
   - `NEXT_PUBLIC_ADMIN_EMAILS`（管理者のメールアドレス、カンマ区切りで複数可）
4. デプロイを実行

---

### 管理者の設定方法

Firestore モードでは **メールアドレスで管理者を指定** する。

**ローカル（`.env.local`）**
```bash
NEXT_PUBLIC_ADMIN_EMAILS=admin@example.com,another@example.com
```

**Vercel**
Vercel ダッシュボード → Settings → Environment Variables に `NEXT_PUBLIC_ADMIN_EMAILS` を追加。

指定したメールアドレスの Google アカウントでログインすると：
- ヘッダーに「マスタ管理」リンクが表示される
- `/admin` ページにアクセスできる

> SQLite モードは常に管理者扱い（ローカル開発用のため制限なし）。
